# デザインガイドライン

このドキュメントはプロジェクトのデザイン決定とその理由を記録します。
将来的な変更時に意図を理解し、悪い方向へ変更してしまうことを防ぐためのものです。

---

## カラーパレット

### Forest（森）テーマ
プロジェクト全体で「癒し」「安心感」「自然」をテーマにしたカラースキームを採用。

```
forest-50:  背景色（最も明るい）
forest-100: カード背景、ボタン背景
forest-200: フッター背景
forest-300: ボーダー
forest-500: プレースホルダー、無効状態
forest-600: 通常テキスト、アイコン
forest-700: 見出し、強調テキスト
forest-800: 最も強調するテキスト
forest-900: メインテキスト
```

**選定理由：**
- 緑系統は心理的に落ち着きと癒しを与える
- メンタルヘルス系アプリに適した配色
- 十分なコントラスト比でアクセシビリティを確保

### アクセントカラー
- `yellow-100/yellow-200/yellow-800`: 「新着」バッジ用
- `green-50/green-200/green-700/green-800`: pain-meter、各種ゲーム要素

**使用禁止事項：**
- 原色の赤（警告以外）
- 高彩度の蛍光色
- forest以外のメインカラー変更

## タイポグラフィ

### フォントファミリー
- **メインフォント**: システムデフォルト（sans-serif）
- **特別な見出し**: `'Kosugi Maru', sans-serif`

**Kosugi Maru使用箇所：**
- ホームページのヒーロータイトル（app/page.tsx:175）
- 理由: 丸みのある優しい印象を与え、癒しのテーマと一致

### フォントサイズ階層
```
text-sm:   小さな補足テキスト
text-base: 通常テキスト
text-lg:   強調テキスト、サブタイトル
text-xl:   バナー、重要な通知
text-2xl:  セクション見出し
text-3xl:  大見出し（モバイル）
text-4xl:  大見出し（デスクトップ）
```

### フォントウェイト
- `font-bold`: 通常の強調（600）
- `font-extrabold`: 最も重要な要素（800）
- `font-semibold`: 中間の強調（600）

**ルール：**
- 見出しは必ずbold以上
- 本文でextraboldは使用しない
- バナーや重要通知のみextrabold可

## レイアウト

### グリッドシステム
```typescript
// ゲーム・記事カード一覧
grid-cols-1         // モバイル
sm:grid-cols-2      // 640px以上
lg:grid-cols-3      // 1024px以上
xl:grid-cols-4      // 1280px以上
```

**理由：**
- Mobile-firstアプローチ
- カードが小さくなりすぎないよう4列まで
- 十分な余白（gap-6）で視認性確保

### 間隔（Spacing）
```
gap-6:  カードグリッド間隔
mb-12:  セクション間マージン
py-8:   メインコンテンツの縦パディング
px-4:   コンテナの横パディング
```

**変更禁止：**
- セクション間のmb-12を減らさない（視認性低下）
- gap-6を4以下にしない（カードが詰まりすぎる）

### コンテナ幅
- `max-w-2xl mx-auto`: ヒーロー説明文（読みやすさ重視）
- `max-w-md mx-auto`: pain-meter（コンパクトに）
- `container mx-auto`: メインコンテンツ

## コンポーネント別デザイン決定

### 1. ホームページ (app/page.tsx)

#### バナーエリア（168-171行目）
```tsx
<div className="text-center mb-6">
  <p className="text-xl font-extrabold text-forest-800">{t("free_now")}</p>
  <p className="text-lg font-bold text-forest-700 mt-2">ブックマーク忘れずに</p>
</div>
```

**デザイン意図：**
- text-centerで中央揃え
- extraboldで「無料」を強調
- mt-2で適度な間隔
- 2行構成で情報を分離

**変更してはいけないこと：**
- font-extraboldをbold以下にしない
- text-xlをlg以下にしない
- mb-6を削除しない（下のコンテンツと被る）

#### ヒーローセクション（173-180行目）
```tsx
<h2 className="text-3xl md:text-4xl font-extrabold mb-4 text-forest-700"
    style={{ fontFamily: "'Kosugi Maru', sans-serif" }}>
  {t("hero_title")}
</h2>
```

**デザイン意図：**
- レスポンシブテキストサイズ（モバイル3xl、デスクトップ4xl）
- Kosugi Maruで柔らかい印象
- mb-4で説明文との間隔確保

**変更してはいけないこと：**
- Kosugi Maruフォントを削除しない
- font-extraboldを弱めない
- レスポンシブサイズ指定を削除しない

### 2. Pain Meter (components/pain-meter.tsx)

**配色：**
- `bg-green-50`: 背景（薄い緑で安心感）
- `border-green-200`: ボーダー（柔らかい境界線）
- `text-green-800`: テキスト（十分なコントラスト）

**形状：**
- `rounded-3xl`: 大きな角丸で優しい印象
- `shadow-md`: 適度な影で浮遊感

**サイズ：**
- `max-w-md mx-auto`: コンパクトに保つ
- `p-6`: 内側に十分な余白

**変更してはいけないこと：**
- roundedをlg以下にしない（優しさが失われる）
- max-w-mdをより大きくしない（横に広がりすぎる）
- green系統から別の色に変えない

### 3. Quiz Card (components/quiz-card/quiz-card.tsx)

**カード構造：**
- ホバー時にshadow-lgで浮き上がる効果
- transition-allでスムーズなアニメーション
- 画像 → バッジ → タイトル → 説明 → メタ情報の順序

**バッジデザイン：**
- 「新着」: `bg-yellow-100 text-yellow-800 border-yellow-200`
- 「ピックアップ」: 別のバッジスタイル

**変更してはいけないこと：**
- ホバー効果を削除しない（UXの重要な要素）
- カード内の要素順序を変更しない（視線誘導が崩れる）

### 4. Featured Quiz (components/featured-quiz/featured-quiz.tsx)

**特徴：**
- グラデーション背景で目立たせる
- より大きなカードサイズ
- 「今週のピックアップ」バッジ

**変更してはいけないこと：**
- グラデーション背景を削除しない（注目度が下がる）
- 通常のQuizCardと同じサイズにしない

### 5. Header (components/header/header.tsx)

**レイアウト：**
- 言語切り替えボタンを右上に配置
- ロゴ・タイトルを左側に配置
- sticky対応でスクロール時も表示

**変更してはいけないこと：**
- 言語切り替えボタンの位置を変えない（慣習に従う）
- stickyを削除しない（ナビゲーション性低下）

### 6. ゲーム個別デザイン

#### Logic Game (components/games/logic/logic.tsx)

**入力画面：**
- 各フィールドに「回答」ボタン（+100pt）
- グリッド表示（最大2セット、横並び）
- 入力時に草エフェクト（🌿）でフィードバック

**ゲーム画面：**
- ホバーベースのマッチング（クリックドラッグより簡単）
- 浮遊する「やらない理由」（行動名 + 理由を表示）
- 下部に横並びの「やる理由」
- 60秒タイマー、スコア表示

**配色：**
- `bg-gradient-to-br from-green-50 to-blue-50`: 爽やかな背景
- 特別な高速理由: ⚡マークで視覚的に強調

**変更してはいけないこと：**
- ホバーベースのマッチングをクリック式に戻さない
- セット最大数を2以上にしない（横表示が崩れる）
- 行動名の表示を削除しない（どのセットか分からなくなる）

#### Expose Game (components/games/expose/expose.tsx)

**特徴：**
- 行動リストに個別の「回答」ボタン
- 回答済みは✓マークで明示
- 各回答で+100pt、confettiアニメーション

**変更してはいけないこと：**
- 回答ボタンのフィードバック（✓、confetti）を削除しない
- ポイント表示を削除しない（モチベーション要素）

## アニメーション

### ConfettiCanvas
- 各ゲームで使用する紙吹雪エフェクト
- ポイント獲得時のポジティブフィードバック

**使用タイミング：**
- フィールド回答完了時
- セット完了時
- ゲーム終了時

**変更してはいけないこと：**
- confettiを削除しない（達成感が失われる）
- 表示時間を極端に短くしない

### Transition効果
- `transition-all`: カードホバー
- `transition-colors`: ボタンホバー
- `duration-300`: 標準アニメーション時間

**ルール：**
- インタラクティブ要素には必ずtransition
- 300ms以下で素早く反応

## レスポンシブデザイン原則

### Mobile-First
すべてのコンポーネントはモバイルを基準に設計

### ブレークポイント
```
sm:  640px  - タブレット縦
md:  768px  - タブレット横
lg:  1024px - 小型ラップトップ
xl:  1280px - デスクトップ
```

### 必須対応
- グリッドは必ずレスポンシブ指定
- テキストサイズはmd:以上で調整
- 余白は画面サイズに応じて調整

**変更してはいけないこと：**
- モバイル表示を無視しない
- ブレークポイントを削除しない

## アクセシビリティ

### コントラスト比
- テキストと背景のコントラスト比は最低4.5:1
- forest-600以上を通常テキストに使用

### フォーカス表示
- すべてのインタラクティブ要素にフォーカススタイル
- キーボードナビゲーション対応

**変更してはいけないこと：**
- コントラスト比を下げない
- フォーカススタイルを削除しない

---

## 禁止事項まとめ

### 絶対に変更してはいけないこと
1. **カラーテーマの大幅変更**（forestテーマは維持）
2. **Mobile-first原則の放棄**
3. **アクセシビリティの低下**
4. **レスポンシブ指定の削除**
5. **ホバー・アニメーション効果の完全削除**
6. **セマンティックなHTML構造の破壊**

### 慎重に判断すべきこと
1. フォントサイズの変更（視認性に影響）
2. 間隔の縮小（窮屈になる可能性）
3. アニメーション時間の変更（UX低下リスク）
4. カード構造の変更（情報階層が崩れる）
